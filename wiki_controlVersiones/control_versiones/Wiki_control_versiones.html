4-. Crea un repositorio en GitLab (GitHub) - Mi repositorio 

Para crear un repositorio en GitHub, debemos ir a la sección repositorios y una vez ahí seleccionar “New”

Escogemos la configuración que nos interese y clicamos en crear.

5-. Sube el proyecto creado anteriormente con el archivo README.md

Para subir el proyecto al repositorio vamos a utilizar la ventana de comandos.

Lo primero será inicializar el repositorio local

git init

A continuación vamos a añadir todos los archivos que queremos subir

git add .

Ahora confirmaremos los archivos y los prepararemos para la subida (commit)

git commit -m “Comentario”

Tendremos que añadir la dirección de nuestro repositorio remoto en GitHub

git remote add origin https://github.com/alexvloure/formacion-plexus

Si queremos comprobar la dirección de nuestro repositorio remoto

git remote -v

Subir los cambios a nuestro repositorio remoto

git push origin master

En la ventana de comandos nos saldrá un mensaje al finalizar la subida.

Comprobamos en nuestro repo que todo se ha subido correctamente.

6-. Instala un gestor de versiones tipo SourceTree o GitExtensions y clona tu repositorio

En mi caso instalaré SourceTree, para ello nos dirigimos a su página oficial. Descargamos la aplicación de escritorio y la instalamos. Nos obligará a crearnos una cuenta en Bitbucket pero luego podremos añadir nuestro repositorio de github, gitlab, etc.

Lo primero que debemos hacer será conectar nuestra cuenta de GitHub (en mi caso). Para ello clicamos en la pestaña remote y seguimos los pasos del siguiente gif. Cuando cliquemos en “Refresh OAuth Token” nos llevará a una página en la cuál debemos dar permisos a sourcetree sobre nuestra cuenta de github.

Para clonar un repositorio debemos situarnos en la pestaña Clone e introducir la ruta del repositorio externo y el local como en el siguiente gif.

7-. Crea la rama master

La rama master ya ha sido creada puesto que al subir el proyecto al repositorio de github debemos crear una rama, y la primera que creamos es la master.

8-. Crea diferentes ramas para cada ejercicio realizado hasta el momento. (Cada ejercicio debe pertenecer a una rama diferente)

Creamos una nueva rama en local para subir el primero de los ejercicios, “formatos-archivos” en mi caso.

Ahora subiremos la rama realizando un commit antes.

Realizamos un push del commit que acabamos de crear.

Debemos hacer este proceso con cada uno de los ejercicios. Una vez realizado el proceso, nuestro repositorio de GitHub debería tener una organización similar a la siguiente.

Tenemos varias ramas y en cada una tenemos uno de los ejercicios que hemos ido realizando hasta la fecha.

9-. Crea una rama develop en la cual deberás incluir las ramas de los ejercicios (ojo, no hay que eliminar las ramas de los ejercicios)

Para esto simplemente debemos crear la rama develop y hacer un merge de todas las ramas de los ejercicios hacia esta.

Ahora hacemos push de la rama develop hacia el repositorio de GitHub

Vemos como el push se ha realizado correctamente y ya tenemos la rama develop en nuestro repositorio externo.

10-. Cada rama tiene que estar en una versión diferente, por ejemplo: o Master v1.1.0 (rama de producción) o Develop v.1.1.1 (rama de desarrolladores)
